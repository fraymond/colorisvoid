# 🦐 《训虾心经》

### ------ OpenClaw 小龙虾从安装到调教的完整实践手册 ------

------------------------------------------------------------------------

## 1️⃣ 安装：从零到可运行的完整流程（Mac 全流程）

> 原则：可控、可调试、可改源码。

### 🔧 环境准备

#### 1. 安装 Homebrew（如果没有）

``` bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

#### 2. 安装 Node.js（建议 v22 以上）

``` bash
brew install node
node --version
```

#### 3. 安装 pnpm

``` bash
npm install -g pnpm
```

------------------------------------------------------------------------

### 📦 下载源码

``` bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

------------------------------------------------------------------------

### 📥 安装依赖

``` bash
pnpm install
```

------------------------------------------------------------------------

### 🚀 全局安装 CLI

``` bash
pnpm add -g .
```

------------------------------------------------------------------------

### 🧭 初始化

``` bash
openclaw onboard
```

如需后台守护：

``` bash
openclaw onboard --install-daemon
```

------------------------------------------------------------------------

### ▶ 启动

``` bash
openclaw dashboard
```

📌 优点：出问题可以直接改源码，而不是黑盒运行。

------------------------------------------------------------------------

## 2️⃣ 初始使用：Dashboard 与多端接入

### 🖥 本地 UI Dashboard

默认运行在：

    http://127.0.0.1:端口号

常见端口：`3000` 或 `5173`

你可以在这里：

-   与小龙虾聊天
-   查看日志
-   管理任务
-   调整记忆

这是最安全的使用方式。

------------------------------------------------------------------------

### 📱 手机远程接入

#### ✅ Telegram（推荐）

优点：

-   API 清晰
-   Bot 接入成熟
-   手机端体验好
-   部署简单

建议搜索：

> 如何创建 Telegram Bot 并获取 Token

------------------------------------------------------------------------

#### ⚠ Discord

优点：

-   可以拉群
-   多个小龙虾交流

风险：

-   配置复杂
-   权限系统繁琐
-   容易被恶意 prompt 注入

不建议公开部署。

------------------------------------------------------------------------

## 3️⃣ 用 AI 编程工具管理小龙虾：Agent 管 Agent

------------------------------------------------------------------------

## 4️⃣ 大模型与成本：Token 是核心成本

### 💰 API 模式

-   每个 token 都计费
-   每次 transaction 都计费
-   编程类任务极度耗 token

一天 40--50 美金并不夸张。

原因：

-   长上下文
-   多轮推理
-   反复修改

------------------------------------------------------------------------

### 🧠 关于 ChatGPT Plus

理论上：

-   Plus ≠ API 额度
-   API 单独计费

但某些 OAuth 集成场景下，可能存在：

-   共享额度
-   非标准调用路径

建议自行在 Usage 页面验证。

------------------------------------------------------------------------

## 5️⃣ 使用心得：真正的训虾之道

### ① 不要一开始给巨大任务

大任务 = 大 token 消耗

建议：

-   先改小函数
-   逐步扩展
-   分阶段推进

------------------------------------------------------------------------

### ② 外部 Skills 谨慎

使用前：

-   看 GitHub Star
-   看 Issues
-   看作者信誉

不要盲目信任。

------------------------------------------------------------------------

### ③ 获取网页知识的方式

#### 🖼 browserrelay

-   通用
-   很费 token

#### 🔍 Brave API

-   结构化
-   token 少
-   容易被网站 block

根据任务选择。

------------------------------------------------------------------------

### ④ 记忆系统

小龙虾的长期记忆通常写在 Markdown 文件中。

建议：

-   写入偏好
-   写入历史决策
-   写入执行记录

你不是在用它，你是在训练它。

------------------------------------------------------------------------

# 🧘 最终心法

真正的训虾之道：

-   控制任务规模\
-   控制 token 消耗\
-   控制技能来源\
-   控制记忆质量

否则，它不是你的助手，而是你的账单生成器。
